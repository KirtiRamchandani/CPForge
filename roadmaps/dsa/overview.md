# DSA Roadmap

## 0. Programming Foundations

Learn input/output, complexity, arrays, strings, functions, recursion, and debugging.

## 1. Arrays And Strings

- What to learn: traversal, frequency maps, prefix sums, two pointers, sliding window.
- Why it matters: most interview problems reduce to array/string invariants.
- Common mistakes: off-by-one, missing empty case, not resetting counters.
- Must solve: Two Sum, Best Time to Buy and Sell Stock, Longest Substring Without Repeating Characters.
- Revision: 1, 3, 7, 14, 30 days.

## 2. Trees

- What to learn: DFS, BFS, recursion state, BST bounds, LCA.
- Why it matters: tests clean state passing and edge cases.
- Common mistakes: null handling, parent handling, recursion depth.

## 3. Graphs

- What to learn: BFS, DFS, connected components, topological sort, Dijkstra, DSU.
- Why it matters: core CP and interview area.
- Common mistakes: not clearing visited, 0/1 indexing, stale priority queue entries.

## 4. Dynamic Programming

- What to learn: state, transition, base case, order, memory optimization.
- Why it matters: separates pattern recognition from brute force.
- Common mistakes: wrong DP state, missing base case, wrong iteration direction.
